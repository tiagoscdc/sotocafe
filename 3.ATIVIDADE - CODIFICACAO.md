## Detalhamento das Escolhas

### Linguagem: TypeScript

**Justificativa:**
- TypeScript é a linguagem recomendada na documentação para desenvolvimento Full Stack JavaScript
- Oferece tipagem estática, melhorando a qualidade e manutenibilidade do código
- Compatível com React.js (frontend) e Node.js (backend)
- Suportado pela maioria das ferramentas e frameworks modernos

### Banco de Dados: PostgreSQL 14+

**Justificativa:**
- SGBD recomendado na documentação do projeto
- Open source e robusto
- ACID compliance garantindo integridade dos dados
- Suporte a JSON para dados flexíveis
- Performance adequada para e-commerce
- Recursos avançados (triggers, stored procedures, views)

**Opções gratuitas para hospedagem do banco:**
- **Supabase** (PostgreSQL gratuito, integração fácil)
- **Railway** (PostgreSQL gratuito, integração com GitHub)
- **Render** (PostgreSQL gratuito, plano free tier)
- **Neon** (PostgreSQL serverless gratuito)

### Hospedagem: Vercel

**Justificativa:**
- **Plano gratuito disponível** com recursos generosos
- **Integração direta com GitHub** - deploy automático a cada push
- **Gera link automático** para cada deploy (ex: projeto.vercel.app)
- Suporte nativo para React.js e Next.js
- Suporte para serverless functions (backend Node.js/Express)
- Deploy instantâneo e preview de branches
- SSL/HTTPS automático e gratuito
- CDN global incluído
- Ideal para projetos acadêmicos e MVP

### Plataforma: React.js + Node.js/Express.js

**Justificativa:**
- Stack Full Stack JavaScript recomendada na documentação
- **Frontend (React.js):**
  - Framework moderno e popular
  - Componentização reutilizável
  - Grande ecossistema de bibliotecas
  - Performance otimizada
- **Backend (Node.js/Express.js):**
  - Mesma linguagem no frontend e backend (JavaScript/TypeScript)
  - Express.js é framework leve e flexível
  - Grande comunidade e bibliotecas disponíveis
  - Adequado para APIs RESTful

### Modo de Codificação: Low-code

**Observação:** 
A documentação técnica do projeto recomenda desenvolvimento tradicional com TypeScript/JavaScript. Se o modo escolhido for Low-code, seria necessário utilizar plataformas como:
- OutSystems
- Mendix
- Bubble
- Power Apps
- Appian

Nesse caso, a linguagem e plataforma seriam gerenciadas pela ferramenta low-code escolhida.
